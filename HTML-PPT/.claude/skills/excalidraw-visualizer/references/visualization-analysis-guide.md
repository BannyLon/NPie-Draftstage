# Visualization Analysis Guide

LLM 分析文章时使用的多角度分析框架。
目标是发现文章中所有适合可视化的结构性内容。

---

## 1. 核心分析流程

输入文本 → 分段扫描 → 识别结构模式 → 匹配图表类型 → 填充结构化数据

每段以"这个内容可以用什么图表表达？"的视角审视。

---

## 2. 图表类型与适用场景矩阵

| 图表类型 | 核心结构 | 典型信号词 | 数据格式 |
|---------|---------|-----------|---------|
| **flowchart** | 线性/分支/循环 | 首先/然后/如果/否则/当/决定 | nodes + edges |
| **comparison** | 并列/对照 | 相比之下/优于/劣势/vs/另一方面 | left_header + right_header + items |
| **tree** | 层级/嵌套 | 属于/分为/包含/子类/层级 | root with children |
| **timeline** | 时序/阶段 | 年月日/Q1/Q2/时期/最初/随后 | events (date + title) |
| **mindmap** | 辐射/关联 | 包括/相关/涉及/例如/方面 | root with branches |
| **cycle** | 循环/迭代 | 循环/反馈/重复/周期/闭环 | nodes + exit_node |

---

## 3. 深度分析框架（逐段扫描）

### 3.1 流程类检测

**检测信号：**
- 序列词：首先、然后、接着、最后
- 条件词：如果、否则、取决于、当...
- 步骤编号：1. 2. 3. / Step 1 / 第一步
- 操作说明：命令式语句
- 因果关系：导致、引发、产生

**示例段落：**
> "用户注册流程：首先填写手机号，然后输入验证码，如果验证码正确则设置密码完成注册，否则重新获取验证码。"

→ 流程图，含判断分支

### 3.2 对比类检测

**检测信号：**
- 对比词：相比之下、vs、不同之处、另一方案
- 优劣词：优势、劣势、优于、不足
- 并列结构：方案A...方案B...
- 表格/列表呈现的横向比较

**示例段落：**
> "方案A采用微服务架构，扩展性好但运维复杂；方案B采用单体架构，开发简单但扩展受限。"

→ 对比图

### 3.3 层级/分类检测

**检测信号：**
- 层级词：属于、分为、包含以下、子类
- 列举结构：一级标题 → 二级项目符号
- 组织关系：CEO → 部门 → 团队
- 分类标记：类型X包括A、B、C

**示例段落：**
> "人工智能分为弱人工智能、通用人工智能和超级人工智能三个层次。其中弱人工智能包括语音识别、图像识别等专用系统。"

→ 树形结构

### 3.4 时间线检测

**检测信号：**
- 时间标记：2024年、Q1、1月、第一阶段
- 演进词：最初、然后、随后、最终
- 里程碑：发布、融资、突破、上线
- 发展阶段：早期、中期、成熟期

**示例段落：**
> "公司于2024年3月成立，6月完成天使轮融资，9月产品首次上线，12月用户突破10万。"

→ 时间线

### 3.5 思维导图检测

**检测信号：**
- 中心词 + 发散结构
- 概念/知识的列举 + 子项
- "包括以下几个方面" + 多个并列子项
- 头脑风暴/知识梳理类内容

**示例段落：**
> "影响用户体验的因素包括：页面加载速度、交互流畅度、视觉设计、内容质量和响应式适配。"

→ 思维导图

### 3.6 循环检测

**检测信号：**
- 循环词：循环、反复、迭代、反馈
- 闭环描述：从A到B再到A
- 改进循环：计划→执行→检查→改进
- 持续过程：持续集成、持续部署

**示例段落：**
> "敏捷开发的核心循环：计划会议确定本轮目标，开发团队编码实现，每日站会同步进度，评审会议检查结果，回顾会议总结经验后进入下一轮计划。"

→ cycle 图

## 4. 复杂段落分析

### 4.1 多图拆分

当一篇文章包含多种结构时，拆分为多个图表：

```
段落1：流程描述 → flowchart
段落2：方案对比 → comparison  
段落3：时间演进 → timeline
```

按文章顺序依次输出 diagrams 数组。

### 4.2 同一段落多结构

如果同一段落同时包含多种结构，选择最突出的：

- 流程 + 时间 → flowchart（时间信息放入节点label）
- 层级 + 对比 → tree（对比信息放入节点描述）
- 如果实在无法取舍 → 拆成两个独立图

### 4.3 隐式结构

有些内容的图表结构不明显，需要推理：

- 一段纯描述性文字中隐藏了因果关系 → 流程图
- 一段叙述中隐含了时间演进 → 时间线
- 一段分类讨论中体现了对比 → 对比图

---

## 5. 质量控制

### 5.1 节点数量限制

| 图表类型 | 建议节点数 | 最大节点数 |
|---------|-----------|-----------|
| flowchart | 3-10 | 20 |
| comparison | 3-8 items | 15 items |
| tree | 5-20 | 40 |
| timeline | 3-10 | 15 |
| mindmap | 5-15 | 25 |
| cycle | 3-6 | 8 |

超过最大节点数时，考虑拆分。

### 5.2 文本长度

- 每个 label 控制在 20 字以内（或 ~30 字符英文）
- 超出应精简或拆分
- 时间线 description 不超过 30 字

### 5.3 一致性检查

- 节点 ID 必须全局唯一
- edge 的 from/to 必须引用存在的节点 ID
- tree/mindmap 的 children/branches 数组可以为空
- cycle 的 exit_node 不可与 nodes 重复
