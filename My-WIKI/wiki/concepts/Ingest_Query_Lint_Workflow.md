# Ingest / Query / Lint 工作流

## 定义

**Ingest / Query / Lint** 是 [[LLM_Wiki]] 模式中定义的三大核心操作，构成了 LLM Wiki 日常运转的完整闭环。这三个工作流覆盖了知识从"进入系统"到"被使用"再到"被维护"的全生命周期。

## 三大工作流详解

### 1. Ingest（摄取）

**目的**：将 `raw/` 中的原始资料编译为 `wiki/` 中的结构化知识页面。

**触发方式**：用户将新资料放入 `raw/`，然后对 LLM 说"请 ingest raw/xxx.md"。

**标准流程**（详细定义见 CLAUDE.md）：

1. 读取上下文 — `wiki/index.md` 和 `wiki/log.md` 的近期记录
2. 通读原始材料，识别关键信息点、核心概念、重要实体
3. 创建 Source Summary 页面（对应源文档的整体摘要）
4. 为主要概念和实体创建或更新 8-15 个独立页面
5. 在页面之间以及与已有页面之间建立 [[Knowledge_Graph_and_Bidirectional_Links]]
6. 更新 `wiki/index.md` 和 `wiki/log.md`
7. 输出变更概览（新建/修改了哪些页面、3-5 条核心洞见）

**关键原则**：
- 绝不修改 `raw/` 中的任何文件
- 优先更新已有页面而非创建重复页面
- 每次 ingest 都应让 wiki 的知识密度和链接密度都有所增加

### 2. Query（查询）

**目的**：基于 wiki 中已编译的知识回答问题。

**标准流程**：

1. 先查 `wiki/index.md`，确定与问题最相关的页面列表
2. 读取相关页面内容，必要时扩展到高度互联的页面
3. 基于 wiki 内容综合回答，使用 `[[页面名]]` 标注信息来源
4. 如果回答过程中产生了有价值的视角或结构，将其**回写**为新的 wiki 页面（synthesis 或 comparison）

**关键原则**：
- 尽量基于 `wiki/` 内容回答，而非直接引用 `raw/`
- 好的回答应该成为 wiki 的永久资产
- 这实现了 [[Compounding_Knowledge]]：查询不只是消耗知识，也在创造知识

### 3. Lint（维护）

**目的**：定期检查 wiki 健康状态，防止知识腐化。

**检查项目**：

| 检查项 | 说明 |
|---|---|
| 矛盾内容 | 不同页面对同一话题的冲突描述 |
| 孤立页面 | 没有入链的"死页面" |
| 缺失概念 | 重要但尚未建立独立页面的概念 |
| 坏链接 | 指向不存在文件的 wikilink |
| 过时内容 | 被新信息取代但未标注的旧观点 |

**标准流程**：

1. 扫描 `wiki/` 中所有页面
2. 生成结构化问题报告（按优先级排序）
3. **在用户确认前不执行大规模修改**——小修正（错别字、断链）可自动完成，大重构需征得确认

## 三者的关系

```
         ┌──────────┐
         │  Ingest  │ ← 新知识进入系统
         └────┬─────┘
              │ 创建/更新页面
              ▼
    ┌─────────────────┐
    │   Wiki 知识库    │ ← 持续增长的知识网络
    └────┬───────┬────┘
         │       │
    ┌────▼──┐ ┌──▼────┐
    │ Query │ │ Lint  │
    └───────┘ └───────┘
    使用知识   维护健康
         │       │
         └───┬───┘
             │ 好的 Query 结果回写为页面
             ▼
        回到 Ingest 循环
```

三者形成一个**增强回路**：
- Ingest 增加知识
- Query 使用并创造知识
- Lint 保证知识质量
- Query 产生的有价值内容触发新一轮 Ingest

## 实践建议

1. **Ingest 一次一个**：虽然可以批量 ingest，但逐个处理并参与讨论能获得更好的质量
2. **Query 完问自己**：这个回答值得保存为 wiki 页面吗？
3. **Lint 定期但不过频**：每周或每两周一次即可，取决于 ingest 频率
4. **让 log 可审计**：每次 ingest 的记录应该清楚到未来你回头看能理解发生了什么
