# 嗯哌AI 视频推广物料生成 Skill

## 概述

这个 skill 用于把一份视频教学解说词（通常是 docx 脚本），转化为可直接发布使用的全套推广物料：多平台标题、推广文案、各平台介绍文案、封面 AI 绘图 Prompt。

这是嗯哌AI（Npie AI / 频道主理人 Banny）频道的标准化产物，已经在几十期真实视频中验证过，使用前必读本文件全部内容，不要凭空发挥偏离已验证的方法论。

---

## 第一步：读懂脚本（不可省略）

收到 docx 脚本后：

1. 用 pandoc 或文档技能转换并完整阅读全文，不要只看开头结尾
2. 提取以下要素，这是后续所有文案的事实基础：
   - **核心工具/事件名称**（必须在标题中体现，是搜索关键词）
   - **核心痛点/认知差**（视频要解决的具体问题，是文案的情绪钩子）
   - **关键数字/结果**（如"10分钟出片""60%孤岛率""省70%时间"，是文案最强的具体支撑）
   - **视频结构**（脚本分几个部分，每部分讲什么，决定YouTube时间轴和介绍文案的段落结构）
   - **视频性质分类**（重要，决定文案语气基调）：
     - 工具教学型（如NotebookLM、Gemini操作教程）→ 干货direct，结构化呈现
     - 趋势/事件解读型（如源码泄露、模型发布）→ 认知冲击，悬念驱动
     - 方法论/工作流型（如三层记忆系统、内容生产逻辑）→ 体系化，强调"为什么"
     - 个人故事/开源分享型（如开源Skill）→ 真诚、第一人称、克制夸张
     - 活泼整活型（如沙雕AI视频、爆款玩法）→ 高能、感叹号、emoji可适度用
   - **是否有具体人名/产品名需要保留**（如"Anijam""Happy Horse""Novi AI"必须原样出现在标题里，不可替换或省略）

3. **如果脚本中的表述存在事实模糊点**（例如"全程三步"实际是两个工具分别三步和四步），在生成前先确认结构，不要把脚本里含糊的表达直接复制进文案放大成错误结论。参考过往案例：用户曾纠正"Claude在当前会话内记得上下文，问题只在跨会话"这类需要精确表述的技术细节。

## 第二步：快速澄清（先问，再生成——这是本skill的强制流程）

读完脚本后，**不要立刻输出全部物料**。先用 `ask_user_input_v0` 工具问 1-2 个关键问题，缩小生成范围，避免后续大改：

**必问其一（受众画像，几乎每次返工都源于此）：**
"这期视频在 [B站/视频号] 的核心受众是哪类人？" 
选项参考：AI工具爱好者 / 内容创作者 / 职场效率党 / 技术极客 / 普通职场人

**必问其二（最深痛点，决定文案钩子是否精准）：**
"他们对 [视频核心主题] 最深的痛是什么？"
选项需要根据脚本内容现场设计，不要用通用选项糊弄——参考过往真实案例的具体颗粒度：
- 不是"想提升效率"，而是"花了很多时间，但老板/甲方总是不满意"
- 不是"不会用AI"，而是"知道X有价值但没有系统方法"
- 不是"想学习"，而是"对着空白页发呆，不知道从哪开始"

如果用户在上传脚本时已经一并说明了受众或语气偏好，可以跳过对应问题，但至少要对模糊点做一次确认性复述（"那这期视频号文案我按xx风格写"）。

如果用户明确说"不用问了，直接生成"或类似表达，跳过本步骤，按脚本内容和下方默认规则直接生成，并在文案中合理假设受众为"嗯哌频道泛AI工具受众"。

## 第三步：生成标题（按平台分别设计，禁止跨平台复用同一逻辑）

三大平台的标题方法论完全不同，必须分别设计，不能用一套逻辑套三个平台。

### YouTube 标题
- **硬性长度限制：≤55字符**（手机端显示约55字符会截断，超出的信息等于不存在）
- 关键词前置：工具名/事件名放最前面（这是搜索引擎和用户扫描的双重需求）
- 中英文混排正常，受众懂英文工具名（如 `Gemma 4` `Ollama` `NotebookLM` 不要翻译）
- 不堆砌标点符号，一个破折号或冒号即可
- 公式参考：`[工具名/事件名]：[核心结果/卖点]，[补充信息]`
- 提供 1 个首选 + 2 个备选

### B站标题
- 长度可以比YouTube宽松（最多80字符，但建议控制在30-40字符内手机端不被截断）
- **核心是情绪/悬念驱动，不是关键词堆砌**——B站用户靠推荐流发现内容，标题要让人停下来
- 至少提供 3 个不同角度的版本，分类标注角度名称，例如：
  - 痛点直击型 / 认知颠覆型 / 结果前置型 / 悬念反差型 / 身份认同型
  - 角度选择要匹配第一步判断的"视频性质分类"
- 如果视频有具体的强反差案例或扎心金句（如"60%是孤岛""你以为是知识图谱，其实是文件夹列表"），优先用这类原话做标题素材，真实感强过精心设计的slogan

### 微信视频号标题
- **必须同时提供长标题和短标题两种**
- 长标题：30-40字符左右，3个版本，风格可以多样化（叙事型/认知型/结果型）
- **短标题：硬性≤16字符**，这是视频号后台的强制限制，超出无法保存。生成后必须逐条数字符数并标注（如"`三个真实武器` — 6字 ✅"），不要想当然认为字数合适
- 短标题至少提供4个版本

## 第四步：生成推广短文案（3个版本，风格区分）

用于封面副标题、朋友圈、社群转发等场景。固定提供3个版本，覆盖不同心理触发点：
1. **痛点/场景代入型**：用具体场景白描开场，让读者"对号入座"
2. **结果/数字冲击型**：用清单或对比直接展示结果
3. **认知/身份升级型**：从更宏观的视角给出价值定位

---

## 第五步：生成三大平台介绍文案（结构、语气、长度各不相同）

### YouTube 介绍文案
结构固定如下，不要省略任何一块：
```
[开场钩子，2-3句，呼应标题]

【核心内容板块，按脚本实际结构分段，用✦或→列出】

[如有具体案例/数据，用具体数字呈现，不要笼统概括]

📌 [资源/Prompt领取方式，如"见置顶评论"]
👍 [点赞引导，结合视频具体内容，不要用通用模板句]
🔔 [订阅引导 + 频道下期预告（如有）]

[完整时间轴，格式：00:00 内容描述]

#标签 #标签 #标签...
```
时间轴要点（重要，曾被用户纠正）：
- 不要逐字复制参考样例的风格，要基于本期脚本实际内容原创设计
- 每个时间点的文字描述要用"动作+结果"结构，不是干瘪的功能名词罗列
- 如果某个环节内部还有子步骤（如"十步流程"），可以在该时间点下用缩进列出子项，不必每个子步骤单独占一行时间轴
- 结尾一行如果有总结性slogan，单独成行，呼应开篇

### B站介绍文案
- 语气比YouTube更口语化、更有"我"的第一人称视角
- 开篇用具体场景或一句扎心提问代入，不要直接罗列功能
- 长度通常比YouTube简介短，重点突出，不需要完整时间轴
- 结尾固定加一个互动提问，引导评论区回复（如"评论区告诉我，你最想先试哪个"）
- 标签用 `#标签` 形式，5-7个

### 微信视频号介绍文案
- **必须提供2个版本**，风格明显区分（例如"痛点共鸣型"和"认知升级型"，或"叙事型"和"观点型"），供用户选择或二次组合
- 篇幅最短，符合视频号用户刷得快的习惯，短句为主
- 如果用户额外要求"粉丝动态文案"，注意这是独立于视频介绍的另一种文体，**通常有严格字符数限制（用户常用230字符），生成后必须实际数字符数确认达标**

## 第六步：生成封面 AI 绘图 Prompt（默认规则见下方，硬编码不需用户重复声明）

### 默认规则（除非用户明确说"这次不需要XX"，否则全部遵守）

1. **主视觉固定为一位年轻漂亮的中国女孩**，每套 Prompt 都要重新描述五官细节以保证生成稳定性（直发或微卷长发、中分、肤白、杏眼、自信微笑等具体描述，不要只写"beautiful Chinese woman"一笔带过）

2. **背景颜色必须明亮鲜艳，严禁纯黑或大面积深色背景**。这是用户多次明确反馈的硬性要求。默认使用暖色调组合（暖金/珊瑚橙/天空蓝/薄荷绿/暖紫罗兰等），在Prompt中明确写出"NO dark tones""maximum brightness""everything glows"等强制性措辞，不能只是隐含。

3. **每套 Prompt 都要把封面文字直接写入画面描述中**（用 `Overlaid bold Chinese typography` 区块呈现具体文案内容），不要只给纯视觉描述让用户自己另外叠字。文字层级固定为：主标题（最大）→ 副标题/卖点 → 底部信息条/标签 → 角标徽章。

4. **数量根据内容复杂度动态决定，3-5套**，原则：
   - 工具教学类（功能点≤3个）→ 3套足够：横版主推（YouTube）、竖版主推（B站/视频号）、强化某个具体场景或案例的版本
   - 多功能盘点类（如"5大插件""12个玩法""6个技巧"）→ 4-5套，需要专门设计一套"全部元素环绕展示"的信息密度版本
   - 强情绪/事件类（如认知颠覆、扎心揭秘）→ 优先设计一套"对比冲击版"（左vs右、装前vs装后、A方法vs B方法）

5. **如果用户提供了真人参考照片**，Prompt 需要：
   - 先用文字精准描述照片中人物的五官特征（不臆测，基于实际看到的）
   - 在 Prompt 末尾加 `--iw 2` 参数（Midjourney图生图权重）
   - 文末补充使用建议：Midjourney（图片链接+--iw 2）/ Dzine AI（先Build Character建角色模型）/ Flux Kontext，三选项都要给

6. **比例规则**：
   - YouTube横版默认 `--ar 16:9`
   - B站/视频号竖版默认 `--ar 3:4`
   - 如果用户要求"视频号视频背景画面"（即剪辑用的分层背景图，不是封面缩略图），结构完全不同，参见下方「特殊产物：视频号分区背景图」

7. 每套 Prompt 都要用英文撰写（AI绘图模型对英文理解更准确），但其中的 Overlaid typography 部分的中文文案要准确、且与本期视频内容强相关，不能用占位符敷衍

8. 末尾固定附「平台使用建议表格」，把每套Prompt对应到最适合的平台和核心视觉卖点

## 特殊产物：视频号分区背景图（按需触发，不要默认生成）

仅当用户明确要求"视频号背景画面""分区背景""上方文字下方视频"这类描述时才生成，结构与封面Prompt完全不同：

- 画布比例固定 `3:4`
- **三段式结构，必须在Prompt开头就声明清楚**：
  - 上方约30-35%：唯一的设计内容区，要让人"不看视频只看这部分就知道讲什么"
  - 中间约45%：完全留白，给16:9视频画面，Prompt中明确写"completely empty, no design"
  - 下方约20%：完全留白，给字幕，同样要求"completely empty"
- 默认走"高级感"路线而非花哨多彩路线（与封面Prompt的"明亮鲜艳"要求不同，这类背景图更适合走杂志编辑风：暖白/暖咖啡色/低饱和度的金色或绿色点缀，克制配色，参考高端杂志排版逻辑——细线分隔、留白充足、字号层级分明）
- 上方设计内容的信息层级固定：品牌角标 → 主标题（大字，可多行错落） → 关键信息标签/分隔线 → 一句话价值陈述
- 同样严禁大面积纯黑，但可以用低饱和度深色文字（如深咖啡棕、深炭灰）作为文字颜色，这与"背景禁黑"不冲突

把以上物料整理为 1-2 份 markdown 文件（标题文案类一份，封面Prompt一份），用 `create_file` 创建到 `/mnt/user-data/outputs/`，文件命名格式：`[视频核心关键词]_推广文案全套.md` 和 `[视频核心关键词]_封面Prompt合集.md`，然后用 `present_files` 交付，并在对话中用2-3句话总结本次产出的核心结构（不要重复粘贴全文内容）。

## 处理常见追加请求（已知模式）

用户经常在首次生成后提出以下追加请求，处理方式如下：

- **"标题超字符了/不够吸引力，重新创意"** → 不要只是重写一条，要先指出具体问题（超了几个字符/平淡在哪），再给3-5条新版本，并标注个人推荐及理由
- **"YouTube标题应该怎么写"（泛化提问）** → 这是在问方法论而非要具体标题，先讲清YouTube标题的底层逻辑（关键词前置+搜索友好+55字符），再给该期示例
- **"重新创意B站/视频号文案，针对性不强"** → 先反问受众画像和最深痛点（回到第二步），拿到答案后再重写，不要凭空猜测就改
- **"封面prompt不够高级/太普通"** → 切换设计语言到"克制配色+精致排版+留白"的编辑感路线，不是简单加更多元素
- **"prompt太长/要在2000字符内"** → 精简但保留核心人物特征描述、关键场景元素、文字层级和比例参数，可删减的是过度形容词堆砌
- **"生成YouTube标签"** → 中文标签和英文标签分别给一组，逗号分隔，覆盖工具名、功能词、品类词、年份等，不需要额外解释
- **"创意一个slogan"** → 字数要求"越少越好"时，给3-5个版本，最短优先，并说明哪个适合做频道长期slogan、哪个仅适用于本期

## 重要原则

1. **永远先理解脚本再动笔**，不要看到关键词就套模板，每期视频的事实细节（数字、案例、工具名）必须准确对应脚本原文
2. **平台逻辑不可混用**：YouTube像搜索引擎使用者写，B站像对朋友说话写，视频号像发朋友圈写
3. **具体数字和真实案例永远比形容词更有说服力**——脚本里如果有具体数字（60%、20分钟、2.6K stars），文案中要优先使用，而不是替换成"很多""效率很高"这类空泛表达
4. **用户的反馈是持续校准信号**，如果用户指出某处事实错误（如"全程三步"实为两阶段），要同步检查标题、三平台文案、封面Prompt是否有同样的错误并一并修正，不要只改用户指出的那一处
