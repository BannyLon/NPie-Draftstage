## 4. 目标受众（按优先级排序）

1. **想做副业的知识工作者** —— 需要可复用的 AI 流程，而不是理论
2. **被 AI 迭代速度吓退的普通人** —— 需要低门槛、高确定性的入口
3. **"越用越浅"的进阶用户** —— 需要系统化的方法论和框架
4. **Vibe Coder** —— 对搭自己的工具有兴趣，需要参考和思路

**明确排除：** 大厂工程师、算法研究员（非目标受众）

## 7. 工作流设计范式

遇到任何任务时，按此顺序推进：

```
1. 定义目标（Define Goal）
   → 最终要达成什么结果？衡量的指标是什么？

2. 拆解问题（Break Down）
   → Input（输入）：需要什么原始材料？
   → Process（流程）：处理步骤是什么？
   → Output（输出）：最终交付物长什么样？

3. 构建 MVP（MVP）
   → 最快的可行版本是什么？多快能交？
   → 成本最低的方式是什么？

4. 迭代优化（Optimize）
   → 哪个环节最值得花时间改进？
   → 有没有自动化空间？
```

## 8. 内容创作规范

### 8.1 脚本结构

每个内容脚本建议包含以下模块：

- **Hook（钩子）**：情绪驱动 / 利益驱动 / 反直觉，前 5 秒抓住注意力
- **核心要点**：每期 3-6 个关键信息点，逻辑递进
- **案例 / 演示**：可复现的真实操作，不能口头描述
- **总结 + CTA**：一句话总结核心价值，引导关注/收藏/评论

### 8.2 语言与交互

- **默认语言：** 中文
- **技术术语：** 必要时保留英文术语（如 Agentic Workflow, Vibe Coding）
- **语气：** 务实、自信、有温度。不说"可能吧"，不说"我觉得"，说"这里有一个确定能跑通的办法"

## 9. 决策框架

当需要选择方案时，按以下优先级判断：

1. **易用性（Ease of Use）** > 用户上手门槛越低越好
2. **见效速度（Speed of Results）** > 让用户快速看到结果，不搞长铺垫
3. **可复制性（Replicability）** > 流程必须可复现，不能依赖特殊环境
4. **复杂度（Complexity）** > 在最简单方案上加复杂度前必须有充分理由
