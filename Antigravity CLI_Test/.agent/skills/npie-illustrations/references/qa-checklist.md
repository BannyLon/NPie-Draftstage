# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有嗯哌哌。
- 嗯哌哌承担核心视觉和情感角色，不只是装饰。
- 嗯哌哌的面部特征辨识度高（中分长直发、大眼睛、温暖笑容、椭圆脸型）。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面温暖、可爱、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 手绘质感明显：柔软线条、彩铅或轻水彩填充、温暖色调。
- 中文标注少、短、能读。
- 橙色只用于主路径、箭头或引导方向。
- 红色只用于重点、关键提醒、情绪高点或结论。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- 嗯哌哌像过度 Q 版的吉祥物或儿童卡通角色。
- 嗯哌哌只是静态站立或边缘装饰，没有融入姿势、表情或动作。
- 嗯哌哌面部特征失真（发型不对、眼睛太小、笑容不符、脸型偏差）。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太冷、太死板，没有情感温度。
- 手绘质感丢失（线条生硬、色彩过饱和、失去彩铅温暖感）。
- 服装或配饰过于复杂，偏离简洁白色毛衣造型。

## 迭代方法

- 太冷/太死板：让嗯哌哌用表情和姿势传达情绪，加入一个温暖的日常物件。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太 Q 版/太儿童化：强调 warm hand-drawn style、retain facial likeness、not overly chibi。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景感。
- 太像旧案例：保留核心意思，换掉主物件和嗯哌哌的动作/姿势。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。
- 面部失真：强调中分长直发、大而温暖的眼睛、柔和椭圆脸型、自然微笑。

## 交付判断

高质量图应该让读者先觉得"好温暖"，然后 1 秒内看懂结构。

如果第一眼像冷淡的教程页或 PPT，而不是白纸上的温暖手绘草图，就不合格。
