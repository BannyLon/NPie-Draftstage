# Obsidian知识库体检：一条Prompt让Claude Code找出所有问题

## 开场钩子

你知道，你的 Obsidian 知识库里有多少笔记是「孤岛」吗？

当我们用 Obsidian 搭建知识库时，一开始的架构往往设计得非常精致。但用着用着，它就会一点点偏离最初的设想：层级越来越多、标签越来越乱、命名越来越随意，最后变成一个「看起来很丰富，实则一团糟」的仓库——而我们自己却常常没有察觉。

这时候，你需要一个「外部视角」。一个能同时扫一眼 100 篇笔记，统计它们的链接密度，找出命名冲突，标记哪些内容重复、矛盾、过时，然后清楚地告诉你：「这里有问题」的工具。

今天，我分享一个超级实用的 Prompt，把它丢给claude code，就可以一键体检你的知识库，快速揪出矛盾、冗余、过期和逻辑漏洞，让你的第二大脑重新变得清晰、有序、好用。

走起！

## 演示 Prompt 的实际效果

你看，这是我运营一段时间的知识库。

里面有碎碎念记录、头脑风暴记录、视频脚本，还有读书笔记。不只是堆文件——我还为这个知识库搭建了三层记忆系统，让 AI 能跨会话记住我的上下文，更智能地辅助我工作。

> 如何搭建三层记忆系统，可以看我之前的这期视频，链接我放在右上角了。

我一直觉得我这个知识库运营得不错——结构清晰、层次分明、笔记之间的互联也做得挺到位。

直到我把这条 Prompt 丢了进去，让它给我的知识库做了一次全盘审计。

结果说实话——比我想的差。

100 篇笔记里，60% 是零入链的孤岛。Daily 文件夹的命名规范——我自己写的——遵守率只有 41%。同一个主题的内容，存在三个不同版本，分散在三个文件里，互相之间没有任何链接。

我以为我搭了一个知识图谱。它告诉我：你那是文件夹列表。

好了，让我们把这条 Prompt 执行后的完整输出拉出来，一起逐段拆开看看它到底有多硬核。

第一步，它会自动扫全 Vault 的文件结构。173 个文件，100 篇用户笔记。

然后它会先读规则。我的 Vault 里每个文件夹都有一个 `instructions.md` 告诉 AI 这里放什么、怎么操作。它在分析内容之前，先把 7 个 instructions 全读了一遍——这就是 「结构优于内容」原则在生效。

接下来，AI 开始进行深度分析。它并行读取关键索引文件、检查 Frontmatter 质量、链接密度以及命名规范。审查完成后，它给我输出了一份完整、坦诚的知识库审查报告。报告首先对我的 Vault 进行了肯定：它说我的知识库拥有一个非常漂亮的骨架，结构搭建得相当用心，尤其是那 7 个 Instructions + Dashboard + Principles，构成了它见过的最用心的 AI 协作元规则系统。紧接着，它直指核心问题：问题主要出在内容层——60% 的笔记是信息孤岛，几乎零入链；Daily 文件夹的命名规则已经崩坏；Script 文件夹里竟然存在 3 份内容高度重复的不同版本；甚至根目录也正在变成新的垃圾场。最后，它用一句话做了总结：你的知识库不是缺少结构，而是缺少执行和持续填充。结构搭好了，却没有把内容持续喂进这个结构里。好消息是，这些问题大多是机械性的，都可以批量修复。

下面，它就针对我的问题进行逐条解读：直接指出我的知识库存在的10条问题。然后给出行动计划：立即要做的、本周可完成的以及需要长期做的。每一项都有具体的命令或操作步骤。

有了这份详细的核查报告，我们就可以逐步的去修复。

## 解构这条 Prompt

好了，到这里你已经看到了这条 Prompt 实际跑出来的效果。现在，让我们反过来拆解一下：这条 Prompt 究竟是怎么驱动 AI 输出如此高质量审查报告的？我把完整的 Prompt 贴在屏幕上，你可以先截图保存。这条 Prompt 整体设计非常精妙，我把它拆成四个核心设计点：

### 第一个设计点：角色定义 + 专业知识框架

这一段不是简单让 AI「扮演」一个专家，而是给它画了一个清晰的专业坐标系。

它明确告诉 AI：你不是普通聊天机器人，而是「Obsidian 知识库资深架构师和重构专家」，同时植入了 PARA、Zettelkasten、Atomic Notes 等最佳实践，以及各种常见痛点。这让 AI 从一开始就站在专业高度，用架构师的视角和标准来审视你的知识库，而不是用通用知识随便应付。

### 第二个设计点：命令它做定量分析 + 点名批评

这条 Prompt 没有让 AI 泛泛而谈，而是强制它进行结构化、定量化的深度审查。它明确列出了 5 大审查重点，从整体结构、碎片化问题、命名规范、链接质量，一直到知识连贯性和量化指标，要求 AI 必须逐项检查，不能遗漏。同时，它还要求 AI「按严重程度排序」，并给出具体笔记路径和例子。这就逼着 AI 不能含糊其辞，必须点名批评，把问题说清楚、说具体。

### 第三个设计点：禁止讨好

这是整个 Prompt 里非常狠的一招。在最后它特别强调："请保持直接、坦诚、专业，不要过度礼貌软化批评。" 这句话直接堵死了 AI 常见的"好好先生"行为模式，防止它为了讨好用户而把严重问题说成"小建议"。正是因为这条指令，AI 才敢真实地指出"60% 是孤岛""根目录正在变成垃圾场"这种直白又扎心的问题。

### 第四个设计点：内置接力棒

这条 Prompt 最聪明的地方在于，它不只是帮你诊断问题，还自带执行接力棒。它会主动给你准备了 1-2 个后续 Prompt，让你能继续让 AI 分批执行重构，不至于一次性信息过载。最后，如果你的 Vault 很大，它还会给出分阶段审查策略，非常贴心。

整个 Prompt 从角色设定、审查重点，到输出格式，全部设计得极其严谨和结构化。它不仅让 AI 帮你找问题，更重要的是，它逼着 AI 给你可执行的解决方案。

## 总结结尾

关于这个 Prompt，你记住三点就够了。

**第一点：** 你的 Vault 对你自己是「雪盲」的。你已经和它相处太久，那些混乱、冗余、孤岛，在你眼里早就变成了「背景」。你看不见的问题，AI 能一眼看出来。它就是一个外部的、专业的、毫不留情的审计视角。

**第二点：** 一条好 Prompt 的价值，不在于它给了你多少建议，而在于它让你真正开始动手。大多数人的知识库不是死于没有方法，而是死于「一直没开始」。这条 Prompt 把启动的摩擦降到了最低——复制、粘贴、回车，你就迈出了第一步。

**第三点：** 这条 Prompt 不挑 Vault 大小。不管你是 100 篇还是 500 篇、1000 篇笔记，都能用。如果量很大，就分批执行——先审 Daily 文件夹，修完再审 Script 文件夹，稳扎稳打。

**最后说清楚边界：** 这条 Prompt 需要 Claude 的 Code Agent / Computer Use 能力（能读文件、能统计、能写文件）。普通 ChatGPT 网页版或基础模型只能给你建议，无法真正执行审查。另外，它也不是「一键魔法」。AI 有时候会建议合并两个「看起来重复」的笔记，但你自己可能有特殊理由保留。所以拿到报告后，一定要自己过一遍再执行。

**Token 消耗方面：** 根据我的实际测试，纯审计阶段大概在 30-40K token 左右，执行阶段取决于你要修改的内容多少。整体属于正常对话的消耗量，不会特别夸张。

## 视频中使用的 Prompt

> 你现在是我的Obsidian知识库资深架构师和重构专家，已帮助多人将混乱的个人知识库转化为高效、连贯的第二大脑。你熟悉Obsidian最佳实践（PARA、Zettelkasten、Johnny Decimal、Atomic Notes、LATCH组织法等），也了解常见痛点：文件夹层级过深/过浅、笔记孤岛（orphans）、重复内容、命名不一致、链接稀疏、MOC缺失、原子性不足等。
>
> 我的知识库已经运营一段时间，现在请你对整个vault进行全盘审查。vault路径/文件夹已提供给你（或通过工具访问），请全面分析。

### 审查重点（必须覆盖）

**1. 整体结构评估：**

- 当前文件夹结构和分类逻辑的优缺点。
- 是否存在不合理的层级（过深导致导航困难，或过浅导致混乱）。
- 推荐的理想文件夹/分类方案（可参考PARA + 项目专区 + 知识领域 + 资源库等），并解释理由。

**2. 分散与碎片化问题（Scattered notes）：**

- 识别主题/概念分散在多个地方的笔记（相同或相似内容未聚合）。
- 建议如何合并、链接或创建MOC（Map of Content）/Hub笔记来集中它们。
- 查找"几乎相同"或重叠的笔记（duplicates或near-duplicates）。

**3. 不合理的地方（Unreasonable/Inconsistent）：**

- 命名约定不一致（标题、文件名前缀、日期格式等）。
- 前言（Frontmatter）使用混乱或缺失关键元数据（tags、aliases、status等）。
- 原子性问题：笔记过长/过杂（应拆分）或过短/无价值（应合并/删除）。
- 链接质量：死链、弱链接、缺少双向链接。
- 孤岛笔记（orphans）和死胡同（无出链/入链的笔记）。
- 工作流断点：如Inbox堆积、未处理的Daily Notes、未归档内容。

**4. 知识连贯性与可发现性：**

- 核心知识图谱是否连贯？哪些领域有明显空白或冗余？
- 搜索/导航效率：推荐的索引、Dashboard、查询或模板改进。
- 长期维护性：哪些习惯会导致未来再次分散？

**5. 量化洞察（如果工具支持统计）：**

- 笔记总数、文件夹分布、平均链接数、未链接笔记比例等。
- 高价值 vs 低价值笔记的识别。

### 输出格式要求（严格遵守，便于我行动）

- **执行摘要**：3-5句总体诊断（最严重的问题是什么？）。
- **优势**：目前做得好的地方（保留并强化）。
- **主要问题**：按严重程度排序，每点列出具体例子（笔记路径/标题）和影响。
- **重构建议**：具体、可执行的行动计划（分优先级：立即做 / 本周 / 长期）。包括新文件夹结构提案、批量处理命令或Dataview查询、命名规则模板等。
- **MOC/Hub推荐**：建议创建哪些枢纽笔记，以及如何迁移内容。
- **下一步Prompt**：提供1-2个后续prompt，让我能让你分批执行重构（例如先处理某个文件夹）。
- **可选**：如果vault很大，建议分阶段审查策略。

> 请保持直接、坦诚、专业，不要过度礼貌软化批评。基于事实和最佳实践说话。如果需要更多具体笔记内容或统计，请告诉我，我会提供。开始审查吧。
